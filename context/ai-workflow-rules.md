# AI Workflow Rules

## Scoping Rules
- Work on one feature or page unit at a time (e.g., Homepage Hero section, then What Vrishtek Does section).
- Do not combine UI changes with complex backend/database implementations in the same iteration.

## Handling the Brand
- **No Hallucinated Boilerplate:** If you need placeholder copy, use the precise phrases provided in `project-overview.md` (e.g., "From architecture to interface, we own the outcome.").
- **Design Honesty:** Ensure the resulting UI matches the dark, premium, technical aesthetic requested.

## Implementing Technical Sections
- When implementing the `/stack` page or writing code blocks for the blog, ensure the content reflects deep, realistic technical competence. Reference PostgreSQL schemas, Prisma configurations, and Next.js specific behaviors accurately.

## Keeping Docs in Sync
- Update `progress-tracker.md` whenever an integration (like Prisma or Resend) is successfully wired up or a core page is completed.
- Ensure any new UI components added via CLI (shadcn) are noted.
